## 数据结构

每条记录包含以下字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `_id` | string | 唯一标识符 (UUID格式) |
| `type` | string | 问题类型 |
| `question` | string | 问题文本 |
| `context` | array | 上下文信息，二维列表结构 |
| `supporting_facts` | array | 支持事实，格式: `[[实体名, 索引], ...]` |
| `evidences` | array | 证据，格式: `[[实体名, 关系, 值], ...]` |
| `answer` | string | 答案 |

## context 结构详解

`context` 是一个嵌套列表，结构为：

```
[
  [实体标题, [描述文本1, 描述文本2, ...]],
  [实体标题2, [描述文本1, ...]],
  ...
]
```

样例：
```json
["Stuart Rosenberg", ["Stuart Rosenberg (August 11, 1927 – March 15, 2007) was an American film and television director...", "He was noted for his work with actor Paul Newman."]]
```

## 数据特点

1. **大规模知识库问答数据集**：包含约16.7万条问答数据
2. **多实体关系型问题**：需要关联多个实体和关系来回答
3. **结构化上下文**：context采用实体+描述的嵌套结构
4. **支持溯源**：每条记录都有supporting_facts和evidences字段用于验证答案
5. **答案类型**：全部为字符串类型（yes/no或具体答案）
